Zero-JS scroll progress bar

The teal bar at the very top is a plain <div>. There is no scroll event listener anywhere on this page — its width is driven entirely by animation-timeline: scroll(root), which ties a regular CSS @keyframes animation's progress to how far the document has been scrolled instead of to time. Scroll down to watch it fill.

Before scroll-driven animations, this exact effect needed a scroll listener, a rAF loop, and manual scrollY / (scrollHeight - clientHeight) math — recomputed on every frame. Here the browser's compositor drives it directly, so it stays smooth even during heavy main-thread work.

Support is landed in Chromium (115+) and Firefox (114+, behind the default now) as of late 2024; Safari is still catching up. On an unsupported browser the bar simply stays static at 0 width — no layout breakage, just a missing enhancement.
